# ProductivityHub

An open-source dashboard that unifies 10 self-hosted productivity tools into a single, beautiful interface. One login, one place to launch everything.

## Quick start

```bash
# 1. Clone
git clone https://github.com/your-org/productivity-hub.git
cd productivity-hub

# 2. Run setup (generates all secrets automatically)
chmod +x scripts/setup.sh
./scripts/setup.sh

# 3. Edit .env
nano .env
#   — Set BASE_DOMAIN to your domain
#   — Set ACME_EMAIL for TLS certificates
#   — Set ADMIN_EMAIL and ADMIN_PASSWORD (your login)

# 4. Add DNS records
#   Point *.yourdomain.com → your server IP
#   (See the DNS table printed by setup.sh)

# 5. Start everything
docker compose up -d --build

# 6. Open your dashboard
open https://yourdomain.com
```

## Architecture

```
Internet
   │
   ▼
Traefik (port 80/443, auto TLS via Let's Encrypt)
   │
   ├── yourdomain.com          → ProductivityHub Dashboard (Next.js)
   ├── cal.yourdomain.com      → cal.diy
   ├── analytics.yourdomain.com → Plausible
   ├── blog.yourdomain.com     → Ghost
   ├── n8n.yourdomain.com      → n8n
   ├── supabase.yourdomain.com → Supabase Studio
   ├── store.yourdomain.com    → Medusa Admin
   ├── docs.yourdomain.com     → AppFlowy
   ├── coolify.yourdomain.com  → Coolify
   ├── campaigns.yourdomain.com → Listmonk
   └── design.yourdomain.com  → Penpot
```

All containers share the `hub` Docker network. No ports are exposed directly — everything goes through Traefik.

## Dashboard features

- **Bento grid overview** — all 10 tools as interactive cards with live status
- **Real-time health monitoring** — polls every 30 seconds, shows latency
- **Embedded iframe view** — launch tools without leaving the dashboard
- **Category filter** — filter by Scheduling, Analytics, Publishing, etc.
- **System dark/light theme** — follows your OS preference, manually overridable
- **Collapsible sidebar** — with per-tool status dots
- **Single sign-on** — one login for the dashboard (each tool has its own auth)

## Configuration

All configuration lives in `.env`. Key variables:

```bash
BASE_DOMAIN=yourdomain.com       # Your domain
ACME_EMAIL=you@example.com       # For TLS cert registration
ADMIN_EMAIL=admin@example.com    # Dashboard login
ADMIN_PASSWORD=changeme          # Dashboard password
NEXTAUTH_SECRET=...              # Auto-generated by setup.sh
```

All other secrets (database passwords, encryption keys) are auto-generated by `./scripts/setup.sh`.

## Updating

```bash
# Pull latest images and rebuild dashboard
docker compose pull
docker compose up -d --build dashboard
```

---

## Adding more tools

1. Add the service to `docker-compose.yml` with Traefik labels
2. Add it to `dashboard/lib/services.ts` (the `SERVICES` array)
3. Rebuild: `docker compose up -d --build dashboard`
